# 04.CrudWithInMemoryData

## Overview

This project teaches a full CRUD flow with Minimal APIs and a simple in-memory repository.

## Learning Objectives

- list, create, update, and delete resources
- use a small repository interface
- return `200`, `201`, `204`, and `404`
- keep REST endpoint names predictable

## Main Features

- `GET /api/tasks`
- `GET /api/tasks/{id}`
- `POST /api/tasks`
- `PUT /api/tasks/{id}`
- `DELETE /api/tasks/{id}`
