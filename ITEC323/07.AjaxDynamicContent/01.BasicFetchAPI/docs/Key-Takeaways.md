# Key Takeaways

- `fetch()` can call local ASP.NET Core endpoints without reloading the page.
- JSON APIs should return clear status codes such as `200 OK`, `201 Created`, `404 Not Found`, and `400 Bad Request`.
- Loading panels and disabled buttons make asynchronous actions easier to understand.
- In-memory services are useful for classroom demos because they remove database setup.
- Polling with `setInterval()` is a simple way to refresh content automatically.
- A development CORS policy prepares students for future cross-origin examples, even when this sample uses same-origin requests.
