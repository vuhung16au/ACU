# Interaction Design Guidelines for AI Agents

This document provides focused interaction design guidance for AI agents building beginner-friendly applications in the ITEC323 repository.

## Overview

Use these guidelines when designing pages, screens, forms, navigation, and feedback for students who are learning application development for the first time.

## Core Interaction Design Principles

### 1. Design for Clarity

- Put understanding before decoration
- Use simple labels that clearly describe the action or content
- Keep page purpose obvious at a glance

### 2. Keep Navigation Predictable

- Use clear, consistent navigation labels
- Keep important actions easy to find
- Avoid surprising navigation patterns that beginners may struggle to follow

### 3. Build Simple Forms

- Keep forms short and grouped by task
- Place related fields together
- Show validation and help text near the relevant field when possible

### 4. Always Provide Feedback

- Show useful feedback for loading, success, and error states
- Make system responses easy to notice and understand
- Help users recover from mistakes with clear next steps

### 5. Focus Each Screen

- Aim for one screen, one purpose
- Avoid overcrowding pages with too many actions or concepts at once
- Leave enough space between sections and controls to improve readability

## Short Policies

- Clarity over decoration
- Predictable navigation
- Useful feedback always
- One screen, one purpose
- Teach patterns students can reuse

## When in Doubt

1. Start with the simplest workable screen
2. Reduce the number of actions shown at one time
3. Make labels and feedback more explicit
4. Refer to the main [AGENTS.md](../../AGENTS.md) for repository-wide guidance
