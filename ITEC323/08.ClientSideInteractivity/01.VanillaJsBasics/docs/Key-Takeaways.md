# Key Takeaways

- `querySelector` and `querySelectorAll` are enough for most beginner DOM tasks.
- `addEventListener` keeps event logic explicit and easy to debug.
- `classList.toggle()` is a clean way to switch UI states.
- Prefer `textContent` for user-provided values and use `innerHTML` carefully.
- Client-side validation should happen before rendering new UI elements.
- Razor Pages and vanilla JavaScript work well together for small interactive features.
