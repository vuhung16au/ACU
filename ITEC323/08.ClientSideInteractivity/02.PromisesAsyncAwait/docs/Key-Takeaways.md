# Key Takeaways

- Promise chains (`then/catch/finally`) and `async/await` are equivalent async strategies.
- `try/catch` with `await` is often easier for beginners to read.
- Always check `response.ok` before assuming a Fetch request succeeded.
- Loading and error panels are essential for user-friendly asynchronous UI.
- JSON APIs with clear status codes help students reason about failures.
- In-memory services are ideal for classroom async demos because setup is minimal.
