# Tailwind CSS Setup Guide for ASP.NET Core Razor Pages

Integrating Tailwind CSS into an ASP.NET Core Razor Pages project is a great way to build modern, responsive UIs quickly. Since Razor Pages uses server-side rendering, the standard approach is to use the **Tailwind CLI via Node.js** to watch and compile your CSS during development.

---

## Prerequisites

- **.NET SDK** installed
- **Node.js and npm** installed (required to run the Tailwind CLI — download from [nodejs.org](https://nodejs.org))

---

## Step 1: Create the Razor Pages Project

```bash
dotnet new webapp -n RazorTailwindApp
cd RazorTailwindApp
```

---

## Step 2: Initialize npm and Install Tailwind

Initialize a `package.json` file and install Tailwind CSS as a development dependency:

```bash
npm init -y
npm install -D tailwindcss
```

Generate the Tailwind configuration file:

```bash
npx tailwindcss init
```

This creates `tailwind.config.js` in the root of your project.

---

## Step 3: Configure Tailwind

Open `tailwind.config.js` and update the `content` array to tell Tailwind which files to scan for CSS class names:

```javascript
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./Pages/**/*.cshtml",
    "./Views/**/*.cshtml",
    "./wwwroot/**/*.html"
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

Tailwind uses these paths to determine which utility classes are actually used — unused classes are stripped from the compiled output, keeping the final CSS file small.

---

## Step 4: Set Up Your Source CSS File

Create a `Styles/` folder in the root of your project and add a file called `app.css`:

```
RazorTailwindApp/
├── Styles/
│   └── app.css      ← your Tailwind source file
├── wwwroot/
│   └── css/
│       └── site.css ← compiled output (generated by Tailwind CLI)
```

Add the three Tailwind directives to `Styles/app.css`:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

> **Why a separate `Styles/` folder?** We keep the source file outside `wwwroot/` so the Tailwind CLI can overwrite `wwwroot/css/site.css` as compiled output without destroying your directives.

---

## Step 5: Add Build Scripts to package.json

Open `package.json` and update the `"scripts"` section:

```json
"scripts": {
  "css:build": "tailwindcss -i ./Styles/app.css -o ./wwwroot/css/site.css --minify",
  "css:watch": "tailwindcss -i ./Styles/app.css -o ./wwwroot/css/site.css --watch"
}
```

| Script | Purpose |
|---|---|
| `css:watch` | Runs during development — recompiles `site.css` instantly on every `.cshtml` save |
| `css:build` | Compiles and minifies CSS for production deployment |

---

## Step 6: Link the Compiled CSS

Open `Pages/Shared/_Layout.cshtml`. The default template already links `site.css` — confirm this tag is present in `<head>`:

```html
<head>
    <link rel="stylesheet" href="~/css/site.css" asp-append-version="true" />
</head>
```

> **Tip:** You can delete the default contents of `wwwroot/css/site.css` — Tailwind will overwrite it on the first build.

---

## Step 7: Run the Application

During development you need **two terminal windows** running simultaneously:

**Terminal 1 — Start the Tailwind watcher:**

```bash
npm run css:watch
```

**Terminal 2 — Run the ASP.NET app:**

```bash
dotnet watch run
```

Now open `Pages/Index.cshtml` and add some Tailwind classes:

```html
@page
@model IndexModel
@{
    ViewData["Title"] = "Home page";
}

<div class="max-w-2xl mx-auto mt-12 text-center">
    <h1 class="text-4xl font-bold text-blue-600 mb-6">Hello Tailwind!</h1>

    <div class="bg-white rounded-lg shadow-md p-6">
        <h2 class="text-xl font-semibold text-gray-800 mb-2">Tailwind is Working!</h2>
        <p class="text-gray-600 mb-4">
            If this card has rounded corners, a shadow, and blue text above,
            your Tailwind integration is successful.
        </p>
        <button class="bg-green-500 hover:bg-green-600 text-white font-medium py-2 px-4 rounded">
            Action Button
        </button>
    </div>
</div>
```

Save the file — the Tailwind watcher compiles the CSS instantly and the browser reloads.

---

## Project File Structure (Final)

```
RazorTailwindApp/
├── Pages/
│   ├── Shared/
│   │   └── _Layout.cshtml
│   └── Index.cshtml
├── Styles/
│   └── app.css              ← Tailwind source (your directives)
├── wwwroot/
│   └── css/
│       └── site.css         ← compiled output (do not edit manually)
├── package.json
├── tailwind.config.js
└── RazorTailwindApp.csproj
```

## Common Issues

| Problem | Solution |
|---|---|
| `npx: command not found` | Ensure Node.js is installed: `node --version` |
| CSS not updating in browser | Check that `npm run css:watch` is still running in Terminal 1 |
| Classes not being compiled | Verify `content` paths in `tailwind.config.js` match your `.cshtml` file locations |
| Large `site.css` in production | Use `npm run css:build` (adds `--minify`) instead of `css:watch` |
